<objective>
Add PWA support to Kings Races so the app is installable on phone home screens and works fully offline.

Purpose: Race officials need the app to load reliably slope-side with no connectivity. PWA precaching ensures the app shell (HTML, JS, CSS) is available offline, complementing the existing localStorage data persistence.

Output: vite-plugin-pwa configured, PWA icons generated, index.html updated with meta tags, production build produces a service worker and web manifest.
</objective>

  <name>Task 1: Install vite-plugin-pwa, generate PWA icons, configure package.json</name>
  <files>package.json, public/pwa-64x64.png, public/pwa-192x192.png, public/pwa-512x512.png, public/maskable-icon-512x512.png, public/apple-touch-icon-180x180.png, public/favicon.ico</files>
  <read_first>package.json, public/favicon.svg</read_first>
  <action>
1. Install vite-plugin-pwa as a dev dependency. Use --legacy-peer-deps because vite-plugin-pwa 1.2.0 declares Vite <=7 as peer dep but works with Vite 8 (per D-03, confirmed via GitHub issue #918):
   ```
   npm install -D vite-plugin-pwa --legacy-peer-deps
   ```

2. Add an overrides entry to package.json to suppress the peer dep warning permanently for future installs:
   ```json
   "overrides": {
     "vite-plugin-pwa": {
       "vite": "$vite"
     }
   }
   ```

3. Install @vite-pwa/assets-generator as a dev dependency for icon generation:
   ```
   npm install -D @vite-pwa/assets-generator --legacy-peer-deps
   ```

4. Generate all PWA icons from the existing public/favicon.svg using the minimal-2023 preset:
   ```
   npx @vite-pwa/assets-generator --preset minimal-2023 public/favicon.svg
   ```
   This produces into public/:
   - pwa-64x64.png (favicon)
   - pwa-192x192.png (manifest icon)
   - pwa-512x512.png (manifest icon, purpose: any)
   - maskable-icon-512x512.png (manifest icon, purpose: maskable)
   - apple-touch-icon-180x180.png (iOS home screen icon)
   - favicon.ico (48x48 ICO fallback)

5. Verify all 6 icon files exist in public/ after generation.
  </action>
