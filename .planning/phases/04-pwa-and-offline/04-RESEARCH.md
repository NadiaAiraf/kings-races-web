# Phase 4: PWA and Offline - Research

**Researched:** 2026-03-29
**Domain:** Progressive Web App (service worker, manifest, installability)
**Confidence:** HIGH

## Summary

This phase wraps the existing Kings Races app in PWA infrastructure so it can be installed to a phone home screen and works fully offline. The app already functions offline via localStorage -- this phase ensures the app shell itself (HTML, JS, CSS) is precached by a service worker so it loads with zero network.

The implementation is straightforward: add `vite-plugin-pwa` to the existing Vite 8 config with `generateSW` strategy, configure a web manifest with app identity, and generate PWA icons from the existing `favicon.svg`. No runtime caching is needed since the app makes no API calls. The main risk is the Vite 8 peer dependency mismatch (vite-plugin-pwa 1.2.0 declares Vite 7 max), which is solved with `--legacy-peer-deps` or an `overrides` entry.

**Primary recommendation:** Use vite-plugin-pwa 1.2.0 with `pwaAssets` integration to auto-generate icons from the existing SVG favicon, and configure `registerType: 'autoUpdate'` for silent background updates.

## Architecture Patterns

### Recommended Configuration Structure

All PWA config lives in `vite.config.ts` -- no separate files needed for this simple use case.

```
public/
  favicon.svg          # Existing -- used as icon source
  pwa-192x192.png      # Generated by assets-generator
  pwa-512x512.png      # Generated by assets-generator
  maskable-icon-512x512.png  # Generated
  apple-touch-icon-180x180.png  # Generated
src/
  ...existing code...
vite.config.ts         # Add VitePWA plugin
index.html             # Add theme-color meta, apple-touch-icon link, update title
```

### Pattern 1: VitePWA Plugin Configuration
**What:** Single plugin call in vite.config.ts handles everything
**When to use:** Always -- this is the only pattern needed

```typescript
// vite.config.ts
import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';
import tailwindcss from '@tailwindcss/vite';
import { VitePWA } from 'vite-plugin-pwa';

export default defineConfig({
  plugins: [
    react(),
    tailwindcss(),
    VitePWA({
      registerType: 'autoUpdate',
      workbox: {
        globPatterns: ['**/*.{js,css,html,ico,png,svg,woff2}'],
      },
      manifest: {
        name: 'Kings Races',
        short_name: 'Kings Races',
        description: 'Race management for Kings Ski Club parallel slalom events',
        theme_color: '#2563EB',
        background_color: '#ffffff',
        display: 'standalone',
        orientation: 'portrait',
        icons: [
          {
            src: 'pwa-192x192.png',
            sizes: '192x192',
            type: 'image/png',
          },
          {
            src: 'pwa-512x512.png',
            sizes: '512x512',
            type: 'image/png',
          },
          {
            src: 'maskable-icon-512x512.png',
            sizes: '512x512',
            type: 'image/png',
            purpose: 'maskable',
          },
        ],
      },
    }),
  ],
});
```

### Pattern 2: PWA Assets Integration (Alternative)
**What:** Use `pwaAssets` option to auto-generate icons at build time
**When to use:** If you want icons generated on-the-fly rather than pre-committed

```typescript
VitePWA({
  registerType: 'autoUpdate',
  pwaAssets: {
    preset: 'minimal-2023',
    image: 'public/favicon.svg',
  },
  manifest: {
    name: 'Kings Races',
    short_name: 'Kings Races',
    description: 'Race management for Kings Ski Club parallel slalom events',
    theme_color: '#2563EB',
    background_color: '#ffffff',
    display: 'standalone',
    orientation: 'portrait',
  },
})
```

**Recommendation:** Use Pattern 1 (pre-generate icons with CLI, commit them). This avoids a build-time dependency on the assets generator and keeps the build fast and predictable. Generate once, commit the PNGs, done.

### Pattern 3: index.html Updates
**What:** Required meta tags and links for PWA
```html
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <meta name="theme-color" content="#2563EB" />
  <meta name="description" content="Race management for Kings Ski Club parallel slalom events" />
  <link rel="icon" type="image/svg+xml" href="/favicon.svg" />
  <link rel="apple-touch-icon" href="/apple-touch-icon-180x180.png" sizes="180x180" />
  <title>Kings Races</title>
</head>
```

### Anti-Patterns to Avoid
- **Custom service worker for a no-API app:** generateSW handles everything. injectManifest adds complexity for zero benefit here.
- **Runtime caching rules:** The app makes no network requests at runtime. Adding runtimeCaching config is dead code.
- **Update prompt UI:** D-04 explicitly says silent auto-update. Do not build a "New version available" toast.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Service worker | Custom SW with fetch/cache handlers | vite-plugin-pwa generateSW | Workbox handles cache versioning, stale entry cleanup, precache manifest hashing |
| Web manifest | Manual manifest.json file | VitePWA manifest option | Plugin generates, hashes, and injects manifest link automatically |
| PWA icons | Manual image resizing | @vite-pwa/assets-generator CLI | Generates all required sizes + maskable variants from one SVG |
| SW registration | Manual navigator.serviceWorker.register() | VitePWA registerType: 'autoUpdate' | Plugin injects registration code and handles update lifecycle |

**Key insight:** vite-plugin-pwa abstracts the entire PWA lifecycle. The only manual work is icon generation (one CLI command) and config in vite.config.ts.

## Common Pitfalls

### Pitfall 1: Vite 8 Peer Dependency Error
**What goes wrong:** `npm install vite-plugin-pwa` fails with ERESOLVE because it requires Vite <=7
**Why it happens:** vite-plugin-pwa 1.2.0 hasn't updated peer deps for Vite 8 yet
**How to avoid:** Use `--legacy-peer-deps` flag or add `"overrides": { "vite-plugin-pwa": { "vite": "$vite" } }` to package.json
**Warning signs:** npm error mentioning "Could not resolve dependency" with vite peer

### Pitfall 2: iOS Cache Eviction
**What goes wrong:** iOS can evict PWA cached assets after ~7 days of inactivity, causing the app to fail to load offline
**Why it happens:** WebKit imposes aggressive storage quotas (script-writable data capped, auto-cleared after inactivity)
**How to avoid:** Not fully preventable. For this app, it's low risk -- the app is used during race events (active use), not dormant for weeks. The precached shell is small (<5MB). If evicted, next online visit re-caches everything.
**Warning signs:** App shows blank screen after not being used for weeks on iOS

### Pitfall 3: Forgetting apple-touch-icon
**What goes wrong:** iOS Add to Home Screen shows a generic screenshot instead of the app icon
**Why it happens:** iOS reads `<link rel="apple-touch-icon">` from HTML, not the manifest icons array
**How to avoid:** Add `<link rel="apple-touch-icon" href="/apple-touch-icon-180x180.png" sizes="180x180">` to index.html
**Warning signs:** App icon on iOS home screen is a webpage screenshot

### Pitfall 4: Title Still Says "my-app"
**What goes wrong:** Browser tab and PWA name show "my-app" instead of "Kings Races"
**Why it happens:** The current index.html has `<title>my-app</title>` (Vite default)
**How to avoid:** Update `<title>` tag to "Kings Races" when modifying index.html

### Pitfall 5: Service Worker Caching Dev Assets
**What goes wrong:** SW caches stale development builds, confusing debugging
**Why it happens:** SW enabled during development
**How to avoid:** Do NOT enable `devOptions.enabled` unless actively testing PWA features. Test PWA behavior with `npm run build && npm run preview` instead.
**Warning signs:** Changes not appearing after page refresh during development

### Pitfall 6: EU iOS PWA Restriction
**What goes wrong:** In EU countries, PWAs may not support standalone mode and open in Safari tabs instead
**Why it happens:** Apple's DMA compliance removed standalone PWA support in the EU
**How to avoid:** Cannot be avoided. Low risk for this project -- Kings Ski Club events are UK-based. Document as a known limitation.
**Warning signs:** "Add to Home Screen" creates a bookmark instead of standalone app in EU
