### Navigation Integration

Phase 3 adds a "Finals" sub-tab to the existing sub-tab navigation. The sub-tabs become:

```
[Teams] [Races] [Score] [Finals] [Standings]
```

"Finals" sub-tab replaces the dedicated scoring view when the discipline phase is `'finals'` or `'complete'`. The sub-tab is always visible but shows a blocked state with explanation text when group stage is incomplete.

### Bracket Matchup Card (D-03, D-04)
- White background, slate-200 border, 16px padding, 8px border-radius (same as RaceCard)
- Top row: placement label ("1st/2nd") in label style (14px semibold), left-aligned
- Seeding context: homeRef and awayRef text in slate-500 label (14px, regular weight), showing e.g. "Winner Group I vs Second Group II"
- Team names: heading style (20px semibold) -- "Team Alpha vs Team Beta"
- If team names not yet resolved (finals not confirmed): show seeding refs only, greyed text
- Scoring buttons: same layout as ScoringFocusView -- two rows of 3 buttons each (Win/Loss/DSQ per team), 64px height
- Scored state: buttons show selected outcome, card gets a subtle left border color (green-600 for winner side)

### Confirm Finals Banner
- Appears at top of Finals sub-tab when all group stage races are scored and no ties remain
- Background: `bg-green-50`, border: `border-green-200`, border-radius: 8px
- Text: heading style, green-800
- "Confirm Finals" button: blue accent, 64px height, full-width, 16px semibold white text
- After confirmation: banner disappears, matchup cards become scorable

### Blocked State Banner
- Appears at top of Finals sub-tab when group stage is NOT complete or ties remain
- Background: `bg-slate-100`, border: `border-slate-200`, border-radius: 8px
- Text: body style, slate-500
- Shows specific reason: either "Complete all group stage races first" or "Resolve tied standings before proceeding to finals"

### Results Table
- Full-width table with columns: Position (#), Team Name, Placement Label
- Semantic `<table>` with `<th scope="col">` headers
- Header row: 14px semibold, slate-500 text, bottom border
- Data rows: 16px body, alternating white/slate-50 backgrounds
- Position column: monospace, 32px width
- Row height: 48px minimum

### Export Button
- Below results table, full-width
- Blue accent background, white text, 56px height
- Label: "Export CSV" with inline download SVG icon (16x16, left of text, 8px gap)
- Disabled state: opacity-40, cursor-not-allowed (when finals incomplete)

## Interaction Contract

### Finals Transition (D-01, D-02)
1. All group stage races scored -> system detects completion
2. If ties exist (hasTies) -> show blocked banner with tie resolution instruction
3. If no ties -> show "Finals Ready" banner with "Confirm Finals" button
4. Official taps "Confirm Finals" -> store transitions discipline phase to `'finals'`
5. Matchup cards resolve team names from group standings and become scorable
6. Confirmation is one-way -- no "unconfirm" action

### Finals Scoring (D-03, FINL-03)
- Same auto-complement logic as group stage: tap Win for one team, Loss auto-selects for other
- Same DSQ handling: if one team DSQ, other must be explicitly tapped (Win only)
- Auto-advance: after recording a finals result, scroll to next unscored finals match
- Matchup cards are independent -- official can score in any order (D-05 free navigation inherited)

### CSV Export (D-05, D-06, EXPR-02)
1. Official taps "Export CSV" button
2. Browser generates CSV blob with all three disciplines
3. Triggers browser download with filename `kings-races-{date}.csv`
4. CSV columns: Position, Team Name, Discipline
5. Disciplines separated by blank row and discipline header row
6. No confirmation needed -- export is non-destructive

### Phase Completion Detection
- When all finals matchups for a discipline are scored -> discipline phase transitions to `'complete'`
- Standings sub-tab shows final results table instead of (or in addition to) group standings
- "Export CSV" button enabled only when at least one discipline is `'complete'`

### Small Event Handling (4-7 teams, single group)
- For events where finals are optional or trivial (e.g., 4 teams: only 1st/2nd and 3rd/4th)
- Finals sub-tab still shows, but with fewer matchup cards
- If no finals defined in cheat sheet (empty finals array): show "No finals for this event size" and skip straight to results

### Destructive Actions

| Action | Confirmation Approach |
|--------|----------------------|
| Confirm Finals | Single-tap action. No undo. Banner text makes the action clear: "Ready to begin finals." The one-way nature is acceptable because group results are preserved and finals can be re-scored. |
| Export CSV | No confirmation. Non-destructive file download. |

There are no destructive actions in Phase 3 that delete data. "Confirm Finals" is a forward-only transition but does not destroy any state.

## Accessibility

Inherited from Phase 2 with Phase 3 additions:

- All interactive elements have visible focus rings (`focus-visible:ring-2 ring-blue-600 ring-offset-2`)
- Touch targets minimum 56px (MOBL-01)
- Scoring buttons use `aria-pressed` to indicate selected state (inherited)
- Finals matchup cards use `aria-label` describing the placement match: `aria-label="Finals match: 1st/2nd place - Team Alpha versus Team Beta"`
- Results table uses semantic `<table>` with `<th scope="col">` headers
- "Confirm Finals" button uses `aria-describedby` linking to the banner text explaining the action
- "Export CSV" button: when disabled, `aria-disabled="true"` with tooltip text in `aria-label`: "Export CSV - complete finals to export results"
- Blocked banner uses `role="status"` to announce state changes to screen readers
- Finals ready banner uses `role="status"` with `aria-live="polite"`
- Color is never the sole indicator -- all outcomes have text labels (inherited)
