**TeamEntryView.tsx** (per D-07, TEAM-01):
- Props: `discipline: DisciplineKey`
- Use `useDisciplineState(discipline)` to get `teams`, `teamCount`, `scores`
- Use `useEventStore((s) => s.setTeams)` to get the `setTeams` action
- Local state: `toastMessage: string | null` for deletion feedback

**Add team logic** (per D-07 "add one at a time", RESEARCH Pattern 3):
- When `onAdd(name)` is called:
  1. Create new team names list: `[...teams.map(t => t.name), name]`
  2. Assign slots using `getCheatSheet` if count >= 4, otherwise sequential:
     ```
     const allNames = [...teams.map(t => t.name), name];
     let newTeams: Team[];
     if (allNames.length >= 4) {
       const structure = getCheatSheet(allNames.length);
       const allSlots = structure.groups.flatMap(g => g.teamSlots);
       newTeams = allNames.map((n, i) => ({ slot: allSlots[i], name: n }));
     } else {
       newTeams = allNames.map((n, i) => ({ slot: i + 1, name: n }));
     }
     ```
  3. Call `setTeams(discipline, newTeams)`
- Per Pitfall 3: ALWAYS use `getCheatSheet().groups.flatMap(g => g.teamSlots)` for slot assignment -- never assume sequential slots for multi-group events.

**Delete team logic** (per UI-SPEC Destructive Actions):
- When `onDelete(slot)` is called:
  1. Find the team name being removed
  2. Filter out the team: `teams.filter(t => t.slot !== slot)`
  3. Reassign slots for remaining teams (same logic as add, using remaining names)
  4. Call `setTeams(discipline, newTeams)`
  5. Set `toastMessage` to `Removed: {teamName}` (per UI-SPEC Copywriting)
- Disable individual team deletion once scores exist (`scores.length > 0`) -- per RESEARCH open question 2. Show only the Reset button in that case.

**Validation:**
- Get max from `getValidTeamCountRange(discipline).max`
- If `teamCount >= max`, disable the input and show validation error "Maximum {max} teams for {discipline}" (per UI-SPEC Copywriting)
- If trying to add with `teamCount < 4`, no error (can keep adding until >= 4)

**Reset discipline** (per UI-SPEC Copywriting):
- Show `ConfirmButton` with:
  - `label="Reset {Discipline}"` (capitalize first letter)
  - `confirmLabel="Confirm Reset?"`
  - `onConfirm`: call `setTeams(discipline, [])` to clear teams (which also clears the structure)
- Only show reset button when `teamCount > 0`

**Team count badge:**
- Below the input: `{teamCount} teams entered` in `text-sm text-slate-500`
- Show minimum notice when < 4: "Minimum 4 teams required" in `text-sm text-amber-600`

**Render structure:**
```
<div className="flex flex-col gap-4">
```
