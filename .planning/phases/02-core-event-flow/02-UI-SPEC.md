# Phase 2 — UI Design Contract

> Visual and interaction contract for the core event flow: team entry, race display, result recording, and live group standings. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind components) |
| Icon library | none (text labels + emoji-free SVG inline icons where needed) |
| Font | system-ui, 'Segoe UI', Roboto, sans-serif (system font stack) |

**Rationale:** STACK.md recommends against heavy component libraries (Bootstrap, MUI) for this mobile-first app. Custom Tailwind utility components keep bundle size minimal and give full control over touch targets and outdoor readability. No icon library needed -- the UI is text-heavy (team names, scores, button labels) with minimal iconography.

---

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between label and badge |
| sm | 8px | Compact element spacing, gap between tab items |
| md | 16px | Default element spacing, card padding, list item gaps |
| lg | 24px | Section padding, spacing between card groups |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page-level vertical padding (top/bottom of views) |

Exceptions:
- Touch target minimum height: 56px (RESL-02, MOBL-01, D-03). All tappable elements use `min-h-14` (56px).
- Scoring buttons use 64px height for gloved use. These are the primary interaction surface and need extra room.

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 16px | 400 (regular) | 1.5 | `text-base font-normal leading-normal` |
| Label | 14px | 600 (semibold) | 1.4 | `text-sm font-semibold leading-snug` |
| Heading | 20px | 600 (semibold) | 1.2 | `text-xl font-semibold leading-tight` |
| Display | 28px | 700 (bold) | 1.15 | `text-[28px] font-bold leading-[1.15]` |

**Font stack:** `system-ui, 'Segoe UI', Roboto, sans-serif` -- fast loading, no external font requests (critical for slope-side offline use).

**Weights used:** 400 (body text, race matchup names), 600 (labels, tab labels, table headers, button text), 700 (display only -- race number in focus mode).

**Monospace:** `ui-monospace, Consolas, monospace` for the score grid cells (3/1/0 values) to maintain consistent column alignment.

This app is used outdoors in bright sunlight with gloves. High contrast is mandatory. Dark mode is deferred (outdoor use = bright screens).

| Accent (10%) | `#2563EB` | `bg-blue-600 text-blue-600` | See reserved list below |
| Destructive | `#DC2626` | `bg-red-600 text-red-600` | DSQ buttons, reset actions only |
| Win | `#16A34A` | `bg-green-600 text-green-600` | Win outcome buttons and indicators |
| Loss | `#D97706` | `bg-amber-600 text-amber-600` | Loss outcome buttons and indicators |
| Text primary | `#0F172A` | `text-slate-900` | Headings, team names, primary content |

**Accent reserved for:**
1. Active discipline tab indicator (underline + text color)
2. Active sub-tab indicator (underline + text color)
3. "Add Team" button (primary CTA during team entry)

**Accent is NOT used for:** general interactive elements, links, or hover states on non-primary actions.

**Outcome colors (Win/Loss/DSQ):** These are the most-tapped elements in the app. Each outcome has a dedicated color that is consistent everywhere it appears (scoring buttons, standings grid cells, result indicators on race cards).

**Contrast ratios (WCAG AA minimum 4.5:1):**

## Layout Contract

### Screen Structure (Mobile Portrait)

```
+------------------------------------------+
|  [Mixed]  [Board]  [Ladies]              |  <- Discipline tabs (top, always visible)
+------------------------------------------+
|  [Teams] [Races] [Score] [Standings]     |  <- Sub-tabs (sticky below discipline tabs)
+------------------------------------------+
|                                          |
|            View Content                  |  <- Scrollable content area
|                                          |
+------------------------------------------+
|  Race 14 of 28                           |  <- Progress bar (bottom, sticky)
+------------------------------------------+
```

### Discipline Tabs (D-04)
- Horizontal row, always visible at screen top
- 3 equal-width tabs: "Mixed", "Board", "Ladies"
- Active tab: blue underline (3px) + blue text
- Inactive tab: slate-500 text, no underline
- Height: 48px, full-width tap target
- Font: 14px semibold

### Sub-Tabs (D-06)
- Horizontal row, sticky below discipline tabs
- 4 tabs: "Teams", "Races", "Score", "Standings"
- Same active/inactive styling as discipline tabs
- Height: 44px
- Scrollable horizontally if needed on very narrow screens (< 320px)

### Team Entry View (D-07)
- Single text input with "Add Team" button beside it
- Input: full-width minus button, 48px height, 16px font, 16px padding
- "Add Team" button: blue accent, 48px height, 14px semibold white text
- Team list below: numbered items (1. Team Name), each row 48px height
- Each team row has a delete icon button (X) on the right, 44px tap target
- Team count badge below input: "8 teams entered" in slate-500 label text

### Race List View (D-08, RACE-03)
- Vertical list of race cards
- Each card: white background, slate-200 border, 16px padding, 8px border-radius
- Card content: Race number (label, slate-500), "Team A vs Team B" (body, slate-900)
- Scored races show outcome indicator (colored dot or badge)
- Round headers ("Round 1", "Round 2") as section dividers, heading style
- Future rounds: visible but greyed out (opacity-40), not tappable

### Scoring Focus View (D-01, D-02, RACE-04, RESL-01, RESL-02)
- Full-width card showing current race
- Race number in display size (28px bold): "Race 14"
- Team A name: heading size (20px semibold), left-aligned
- Team B name: heading size (20px semibold), left-aligned
- Three buttons per team in a row: [Win] [Loss] [DSQ]
  - Button height: 64px
  - Button width: equal thirds minus 8px gap
  - Button font: 16px semibold, white text
  - Button colors: green-600 (Win), amber-600 (Loss), red-600 (DSQ)
  - Unselected state: bg-slate-100, text-slate-600 (outlined feel)
  - Selected state: filled with outcome color, white text, subtle ring
- After recording both teams: auto-advance to next unscored race (D-02)
- "Undo" text button below the scoring area: 14px, slate-500, 44px tap target
- Progress: "Race 14 of 28" below race card

### Standings View (D-09, D-10, STND-01, STND-02, STND-03)
- Full-screen overlay toggle (not a modal -- replaces content area)
- Toggle button: "View Standings" / "Back to Scoring" in sub-tab area or floating
- Table layout per group:
  - Group header: "Group A" in heading style
  - Columns: Rank | Team | W | L | DSQ | Pts | Race Results Grid
  - Race results grid: small colored cells (24x24px) showing 3/1/0 per race
  - Cell colors match outcome colors (green-100, amber-100, red-100)
  - Tie indicator: amber warning icon or "TIE" badge next to tied teams
- Table text: 14px for cell data, 14px semibold for headers
- Alternating row backgrounds: white / slate-50

## Interaction Contract

### Result Recording (RESL-02: 2 taps max)
1. Tap outcome for Team A (e.g., "Win") -- button fills with green
2. Team B auto-selects the complementary outcome (if A wins, B loses) -- user confirms or overrides
3. Total: 1 tap to record (auto-complement) or 2 taps to override

**Auto-complement logic:**
- Team A = Win -> Team B auto-selects Loss
- Team A = Loss -> Team B auto-selects Win
- Team A = DSQ -> Team B must still be tapped (could be Win or DSQ, ambiguous)
- Team B = DSQ -> Team A must still be tapped (same reason)

### Auto-Advance (D-02)
- After both teams have outcomes recorded, wait 300ms then scroll to next unscored race
- No confirmation dialog, no animation beyond scroll
- If all races in current round are scored, advance to next round header

### Undo/Correct (RESL-03, RESL-04)
- Each scored race card shows a small "Edit" link (14px, slate-500)
- Tapping "Edit" re-opens the scoring buttons for that race, pre-filled with current result
- Tapping a different outcome immediately replaces the old one
- Standings recalculate on every score change (derived state, not stored)

### Discipline Switching (D-04, D-05, TEAM-03)
- Tapping a discipline tab switches all sub-views to that discipline
- Current sub-tab selection persists across discipline switches
- No data loss -- each discipline maintains independent state

## Accessibility

- All interactive elements have visible focus rings (`focus-visible:ring-2 ring-blue-600 ring-offset-2`)
- Touch targets minimum 44x44px (W3C) with 56px preferred (MOBL-01)
- Scoring buttons use `aria-pressed` to indicate selected state
- Discipline and sub-tabs use `role="tablist"` and `role="tab"` with `aria-selected`
- Standings table uses semantic `<table>` with `<th scope="col">` headers
- Color is never the sole indicator -- outcome buttons always include text labels ("Win", "Loss", "DSQ")
- Progress bar uses `role="progressbar"` with `aria-valuenow` and `aria-valuemax`
