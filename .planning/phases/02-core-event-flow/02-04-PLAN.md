**Auto-complement logic** (per D-01, RESEARCH Pattern 2):
```typescript
function getComplement(outcome: RaceOutcome): RaceOutcome | null {
  if (outcome === 'win') return 'loss';
  if (outcome === 'loss') return 'win';
  return null; // DSQ has no auto-complement
}
```

When Team A outcome is tapped:
1. Set `homeOutcome` to the tapped outcome
2. Calculate complement: `const complement = getComplement(outcome)`
3. If complement exists, set `awayOutcome = complement` and immediately commit the score
4. If complement is null (DSQ tapped), wait for Team B tap

When Team B outcome is tapped:
1. Set `awayOutcome` to the tapped outcome
2. If `homeOutcome` is already set, commit the score
3. If `homeOutcome` is null, calculate complement and auto-set homeOutcome, then commit

**D-01 constraint enforcement** (per Pitfall 6):
Valid outcome pairs: Win/Loss, Loss/Win, Win/DSQ, DSQ/Win.
- When one team selects DSQ, the other team's available options are ONLY Win (disable Loss and DSQ)
- Implement: `const disabledOutcomes` logic per team based on the other team's selection:
  ```
  If otherTeam === 'dsq': disable 'loss' and 'dsq' for this team (only Win allowed)
  If otherTeam === 'win': disable 'win' for this team (only Loss or DSQ allowed)
  If otherTeam === 'loss': disable 'loss' and 'dsq' for this team (only Win allowed)
  ```

**Committing a score:**
```typescript
function commitScore(home: RaceOutcome, away: RaceOutcome) {
  const raceId = `r1-${activeRace.raceNum}`;
  recordResult(discipline, {
    raceId,
    homeSlot: activeRace.homeSlot,
    awaySlot: activeRace.awaySlot,
    homeOutcome: home,
    awayOutcome: away,
  });
  // Reset local state immediately
  setHomeOutcome(null);
  setAwayOutcome(null);
  setEditingRaceId(null);
}
```

**Auto-advance** (per D-02 -- IMMEDIATELY, NO DELAY):
- CRITICAL per D-02: "auto-advance immediately. No delay."
- Do NOT use setTimeout. Do NOT add any artificial delay.
- After calling `recordResult`, immediately reset local state (homeOutcome, awayOutcome, editingRaceId to null).
- The `useCurrentRace` hook will synchronously recompute on the next render because the Zustand store update triggers a re-render. The view will naturally show the next unscored race.
- This means: call `commitScore()` which calls `recordResult()` and resets local state. React re-renders. `useCurrentRace` returns the new next-unscored race. Done. No delay needed.
